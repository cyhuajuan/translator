# Coding Conventions

**Analysis Date:** 2026-03-27

## Tooling

**Formatter & Linter:** Biome
- Version: `2.4.8`
- Config: `biome.json`

**Formatting Rules:**
- Indent style: `space`
- Indent width: `2`
- Quote style (JS/TS): `single`
- Quote style (JSX): `double`

**Linting:**
- Biome linter with `recommended` rules enabled
- `assist.enabled: true` for source actions

## Naming Patterns

**Files:**
- React components: `PascalCase.tsx` (e.g., `button.tsx`)
- Utilities: `camelCase.ts` (e.g., `utils.ts`)
- Routes: `kebab-case.tsx` (e.g., `index.tsx`, `__root.tsx`)
- Generated files: `*.gen.ts`

**Functions:**
- CamelCase for functions (e.g., `createRootRoute`, `RouteComponent`)
- PascalCase for React component functions (e.g., `Button`)
- Descriptive names - no abbreviated names

**Variables:**
- CamelCase for variables
- Type annotations preferred for function parameters and return values

**Types:**
- PascalCase for types and interfaces (e.g., `ClassValue`, `VariantProps`)

## Code Style

**Imports:**
- Named imports preferred: `import { Route } from '@tanstack/react-router'`
- Type imports when only types: `import { type ClassValue } from 'clsx'`
- Default imports for React: `import React from 'react'`
- Path alias `@/` for src directory (configured in `tsconfig.json`)

**Import organization:**
```typescript
// External
import { createFileRoute } from '@tanstack/react-router';
// Internal (path alias)
import { cn } from '@/lib/utils';
```

**Component Patterns:**
```typescript
// Button component pattern from src/components/ui/button.tsx
import * as React from "react"
import { cva, type VariantProps } from "class-variance-authority"
import { Slot } from "radix-ui"
import { cn } from "@/lib/utils"

const buttonVariants = cva(...)

function Button({
  className,
  variant = "default",
  size = "default",
  asChild = false,
  ...props
}: React.ComponentProps<"button"> &
  VariantProps<typeof buttonVariants> & {
    asChild?: boolean
  }) {
  const Comp = asChild ? Slot.Root : "button"
  return <Comp data-slot="button" {...props} />
}

export { Button, buttonVariants }
```

**Utility Patterns:**
```typescript
// src/lib/utils.ts
import { clsx, type ClassValue } from 'clsx';
import { twMerge } from 'tailwind-merge';

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

## Error Handling

**Type Safety:**
- Strict TypeScript mode enabled (`strict: true`)
- `noUnusedLocals: true`
- `noUnusedParameters: true`
- `noFallthroughCasesInSwitch: true`

**Generated Code Exclusions:**
- Files like `routeTree.gen.ts` are excluded from linting
- Use `// @ts-nocheck` or `// eslint-disable` sparingly when needed

## Path Aliases

**Configured in `tsconfig.json`:**
```json
{
  "baseUrl": ".",
  "paths": {
    "@/*": ["./src/*"]
  }
}
```

**Configured in `vite.config.ts`:**
```typescript
resolve: {
  alias: {
    '@': path.resolve(__dirname, './src'),
  },
}
```

**shadcn components.json aliases:**
```json
{
  "aliases": {
    "components": "@/components",
    "utils": "@/lib/utils",
    "ui": "@/components/ui",
    "lib": "@/lib",
    "hooks": "@/hooks"
  }
}
```

## React Patterns

**Functional Components Only:**
```typescript
function RouteComponent() {
  return <div>Hello "/"!</div>;
}
```

**With TypeScript Props:**
```typescript
interface Props {
  name: string;
}

function Component({ name }: Props) {
  return <div>{name}</div>;
}
```

**TanStack Router Pattern:**
```typescript
export const Route = createFileRoute('/')({
  component: RouteComponent,
});
```

## Tailwind CSS

**Class Organization:** Tailwind classes in `className` with shadcn `cn` utility for conditional classes

**Styling Pattern:** CVA (class-variance-authority) for component variants
